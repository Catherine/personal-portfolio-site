Pixels and code

I am both a developer and a designer. I aspire to build interfaces that provide the user with a seamless experience. Additionally, I believe that code is useless if it can only be understood by the writer. Through explanatory comments, format consistency and refactoring, I strive to write code that can be utilized and reused by others.

Driven by passion

A design may be beautiful and innovative, but it isn't any good if it doesn't fulfil the need of a user. A product must work as good as it looks. I am passionate about learning how users interact with computers and technology in order to improve interfaces and users' experiences.

Social Proof Widget

Hypothesis: When users hear of or see other people using a product to solve needs similar to them, they are more willing to use the same product.

What we did: Designed and created two front-end widgets (AB tested) using HTML5, Javascript, and CSS using Design for Delight methodologies. Social posts were pulled from Twitter, HouseParty, and our reviews using a back-end Java service with JAX-RS and AWS Database.

Web Design and Development

This was my final project for IM 4100 Web Design and Development during the fall of 2013. The class emphasized tools and strategies for design, such as site maps, wireframes, prototypes, usability testing, and iterative development.

My project explored parallax scrolling, single-page navigation challenges and solutions, designing images and assets, and creating an online educational experience about the clothing production industry.

Can be found at: catherinepatchell.com/wd

PrereqMe

Planning out courses is frustrating and tedious. My team built a web application called PrereqMe to solve this problem and improve the process while taking CCIS's Human Computer Interaction course. Our product was able to help students visually view prerequisites, provide convenient access to course descriptions, and help them plan for the future.
